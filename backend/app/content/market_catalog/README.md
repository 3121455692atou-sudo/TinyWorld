# 集市物品库格式规范

这个目录暂时只提供数据和独立后端服务，不会自动接入任何世界观、地点或工具注册表。

## 文件

- `items.json`: 集市物品库。其他 LLM 扩充物品时只改这个文件。

## 顶层结构

```json
{
  "version": 1,
  "items": []
}
```

## 通用字段

每个物品都必须有这些字段：

- `item_id`: 稳定唯一 ID，建议用 `food.xxx` 或 `item.xxx`，只使用小写字母、数字、点、下划线、短横线。
- `kind`: 只能是 `food` 或 `item`。
- `name`: 展示给角色和前端看的中文名。
- `aliases`: 搜索别名数组，可为空。
- `description`: 展示介绍。食物和物品都要写。
- `tags`: 搜索和礼物偏好的标签数组，建议包含品类、口味、材质、用途，如 `茶`、`甜食`、`工具`。
- `price`: 非负整数。
- `gift_tags`: 用于送礼判定的标签数组，可复用 `tags`，但建议更偏向偏好，如 `可爱`、`实用`、`苦味`。

## 食物字段

`kind` 为 `food` 时必须额外填写：

- `satiety_delta`: 食用后增加的饱食度。
- `mood_delta`: 食用后作为情绪价值的增减。
- `spoil_after_hours`: 腐败小时数。当前集市食物统一写 `24`，即一天不吃会腐败。

食物示例：

```json
{
  "item_id": "food.matcha_mochi",
  "kind": "food",
  "name": "抹茶麻薯",
  "aliases": ["抹茶", "麻薯", "茶点"],
  "description": "软糯的麻薯裹着微苦的抹茶粉，甜味收得很轻。",
  "tags": ["茶", "抹茶", "甜食", "点心"],
  "gift_tags": ["茶", "抹茶", "甜食", "柔软"],
  "price": 8,
  "satiety_delta": 18,
  "mood_delta": 4,
  "spoil_after_hours": 24
}
```

## 普通物品字段

`kind` 为 `item` 时不要填写食物字段。普通物品不会腐败，只需要名字、介绍、标签、价格和礼物标签。

物品示例：

```json
{
  "item_id": "item.cotton_towel",
  "kind": "item",
  "name": "棉毛巾",
  "aliases": ["毛巾", "洗漱用品"],
  "description": "一条吸水性不错的浅色棉毛巾，边角缝得很整齐。",
  "tags": ["日用品", "清洁", "布料"],
  "gift_tags": ["实用", "清洁", "柔软"],
  "price": 10
}
```

## 搜索规则

后端会在 `name`、`aliases`、`tags`、`gift_tags` 里做包含匹配。查询词和字段互相包含都算命中，所以：

- 搜 `抹茶` 会命中名字或别名里有 `抹茶` 的物品。
- 如果物品有 `茶` 标签，搜 `抹茶` 也会命中，因为 `茶` 包含在查询词里。
- 搜 `茶` 会命中所有名字、别名或标签里带 `茶` 的物品。

## 送礼偏好

角色偏好暂存于 `agent.desires_json["gift_preferences"]`：

```json
{
  "likes": ["茶", "甜食", "实用"],
  "dislikes": ["苦味", "廉价塑料"]
}
```

送礼时后端用物品的 `name`、`aliases`、`tags`、`gift_tags` 和目标偏好匹配。命中喜欢会增加好感，命中讨厌会减少好感；同时命中时讨厌优先。
