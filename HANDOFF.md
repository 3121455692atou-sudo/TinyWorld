# 修改记录

## 2026-06-09

- 清理根目录交接/修复报告文件，只保留 `HANDOFF.md` 和 `HANDOFF.en.md`。
- 一键配置、人员配置导入导出、历史配置复用会保留初始认识与好感设置。
- 历史人员配置导出会从身份认知和关系好感生成可复用的初始认识配置。
- 狼人杀清晨公告增加去重，避免同一天重复写入告示牌和尸体发现公告。
- 同一世界的模拟 step 串行执行，避免自动运行和手动继续并发推进导致重复事件。
- 狼人杀告示牌文案改为“狼人存在于村中”。
- 狼人杀自动售货机接入窄范围商品查询、推荐、购买和背包补给使用工具。
- 集市/自动售货机商品查询文案改为中性“询问/可购买”，避免固定摊主叙述泄漏到其他场景。
- 食物和饮品统一走背包补给使用工具，根据物品属性决定吃或喝。
- LLM 连续失败后的手动开始/继续会重置失败计数，允许再次请求同一 agent 的 LLM。
- 历史身份库前端提升检索和筛选体验。
- 运行设置、一键配置总览、导入导出与复用入口在前端保持一致展示。

验证：

- `uv run pytest backend/app/tests -q` 通过。
- `npm --prefix frontend run build` 通过。

## 2026-06-07

- 新增独立集市物品目录与背包物品服务，支持搜索、推荐、购买、食用/饮用、放置、捡起、移交和送礼好感判定。
- 普通模拟不再在 LLM 选择前强制代选急迫生存工具；饥饿、口渴、疲劳只影响提示词和菜单优先级。
- 移除普通模式 LLM/修复失败后的固定 fallback 行动；模型没有给出可执行行动时不再生成固定台词或替代行动。
- 工具执行格式和目标类失败会给出纠正提示，并在重复失败后暂时从本轮菜单隐藏对应工具。
- 普通说话、求助、介绍、社交、犯罪对质等行动不再由后端补默认角色台词，缺正文会交给验证器反馈。
- 狼人杀被放逐者会被写入公开记忆，并从后续发言、投票、夜袭、查验、守护目标中排除。
- 狼人杀夜袭胜负判定后不再推进到下一天，改为进入最终发言队列。
- 狼人杀最终发言移除固定兜底台词，空输出或疑似截断不会写入事件流。
- 狼人杀第 1 天白天只自由交流，第 1 天夜晚开始夜间能力，第 2 天开始正式圆桌和投票。
- 狼人杀圆桌、投票、夜间阶段使用更明确的主持流程和角色工具，减少普通行动抢走阶段回合。
- 前端事件、左侧快照、头像、地点人数和 WebSocket 刷新逻辑做了分批与节流优化。
- 前端影响世界面板、模型/身份配置、TTS、导入导出、配置总览等 UI 做了整理。
- OpenAI-compatible 客户端支持流式响应、生成参数和并发限制。
- Windows 启动脚本优先复用已有 `uv`，找不到时再安装。

验证：

- 后端 pytest 测试套件通过。
- 前端构建通过。
