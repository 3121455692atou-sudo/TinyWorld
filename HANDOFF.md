# 修改记录

## 2026-06-09

- 一键配置、人员配置导入导出、历史配置复用会保留初始认识与好感设置；历史人员配置导出会从身份认知和关系好感生成可复用的初始认识配置。
- 同一世界的模拟 step 串行执行，避免自动运行和手动继续并发推进导致重复事件。
- 狼人杀清晨公告增加去重，避免同一天重复写入告示牌和尸体发现公告。
- 狼人杀告示牌文案统一为“狼人存在于村中”，用于公开说明狼人仍然存在；狼人消失后由胜负判定处理阵营胜利。
- 狼人杀自动售货机接入窄范围商品查询、推荐、购买和背包补给使用工具。
- 集市/自动售货机商品查询文案改为中性“询问/可购买”，避免固定摊主叙述泄漏到其他场景。
- 食物和饮品统一走背包补给使用工具，根据物品属性决定吃或喝。
- LLM 连续失败后的手动开始/继续会重置失败计数，允许再次请求同一 agent 的 LLM。
- 历史身份库前端提升检索和筛选体验。
- 运行设置、一键配置总览、导入导出与复用入口在前端保持一致展示。

验证：

- `uv run pytest backend/app/tests -q` 通过。
- `npm --prefix frontend run build` 通过。

## 2026-06-07

- 新增独立集市物品目录与背包物品服务，支持搜索、推荐、购买、食用/饮用、放置、捡起、移交和送礼好感判定。
- 普通模拟不再在 LLM 选择前强制代选急迫生存工具；饥饿、口渴、疲劳只影响提示词和菜单优先级，具体行动仍由 LLM 从菜单选择。
- 移除普通模式 LLM/修复失败后的固定 fallback 行动；模型没有给出可执行行动时不再由后端合成固定台词、移动、休息、求助或 `do_nothing`。
- provider/API 请求失败才累计 `llm_consecutive_failures` 并触发自动暂停；模型有回复但行动头或格式不可解析只记录协议错误，不再当成 LLM 崩溃。
- 工具执行格式、目标、地点类失败会给纠正提示；重复失败后，本轮菜单会暂时隐藏对应工具，减少同一错误反复刷屏。
- 普通说话、求助、介绍、社交、犯罪对质等行动不再由后端补默认角色台词；缺少正文会交给验证器反馈给 LLM。
- 需要台词的普通工具补齐菜单要求，包括自我介绍、拒绝介绍、强制社交、成年边界请求、犯罪对质等。
- 狼人杀被放逐者会被写入公开记忆，并从后续发言、投票、夜袭、查验、守护目标中排除。
- 狼人杀夜袭结算触发胜利后会保留 `winner` 和最终发言队列；阶段同步不再继续推进到下一天或生成新的清晨尸体事件。
- 狼人杀最终发言移除固定兜底台词；空输出、错误输出或疑似截断不会写入事件流，也不会推进最终发言队列。
- 狼人杀最终发言支持更完整的终局发言，并检测 provider 返回的截断原因，避免把半截文本当作成功结果。
- 狼人杀第 1 天白天只自由交流，第 1 天夜晚开始夜间能力，第 2 天开始正式圆桌和投票。
- 狼人杀圆桌、投票、夜间阶段使用更明确的主持流程和角色工具，减少普通行动抢走阶段回合。
- Windows 启动脚本优先复用 PATH 里的 `uv`；找不到时再尝试安装，后续依赖同步和后端启动共用同一个 `uv` 命令。

验证：

- 后端相关回归测试通过。
- 前端构建通过。

## 2026-06-06

- `/api/worlds` 列表接口不再为了列表展示而写入运行状态，也不再返回完整世界 settings，减少前端列表读取阻塞。
- SQLite 下 `parallel` 请求模式不再让同一轮多个角色各开独立 Session 并发提交，改为顺序选择行动后批量执行，避免 `database is locked`。
- 左侧轻量快照改为读取最新事件时间，避免事件已推进但侧栏时钟停在旧世界时间。
- 高频 `left-snapshot` 不再返回完整世界 settings 和头像 base64；前端保留当前完整 settings，并按世界补水一次完整头像。
- 前端 GET 请求不再默认携带 `Content-Type: application/json`，避免开发模式高频 GET 触发 CORS 预检。
- 左侧居民、地点、头像、世界时间合并逻辑加强，避免轻量快照把已有头像或配置覆盖成空值。
- 关系阶段路由新增统一计算：关系推进、确认交往、分手、修复关系、成年亲密、避孕、验孕等工具会按具体目标的好感、信任、熟悉度和关系状态过滤排序。
- `validate_tool` 增加关系阶段硬校验，防止 LLM 绕过菜单直接调用当前关系阶段不该开放的关系/亲密工具。
- 成年亲密语义自动对齐不再只因为世界开启生育系统就触发；必须当前菜单开放且目标关系阶段允许。
- 狼人杀地点与提示弱化隐藏身份明牌信息；居民触发规则前只知道普通村庄设定。
- 狼人获得同伴私有记忆；夜袭需要同伙目标一致，目标不一致会要求狼人先密会统一。
- 工具候选和验证加强：隐藏系统工具、狼人专用工具、目标参数、已知姓名路由更严格，减少普通居民看到秘密工具或身份信息。
- 公共事件流隐藏低信息事件；`do_nothing`、工具失败占位和候选工具调试事件不再污染普通事件流。
- 狼人杀默认节奏调整为 08:00 自由交流、12:00 圆桌、18:00 投票、22:00 夜间行动。
- 多狼人夜晚改为先强制狼人密会；所有存活狼人至少完成一次密会后才开放夜袭。
- 狼人杀胜负判定不再立刻结束世界，而是写入胜负已定事件并进入最终发言队列；所有最终发言完成后才写游戏结束。
- 前端初始化读取本地世界列表增加超时兜底，后端无响应时不再无限停在 loading。

验证：

- 后端编译通过。
- 相关后端回归测试通过。
- 前端构建通过。

## 2026-06-05

- 公开事件文本在后端创建、API 序列化和前端渲染三层做清洗，内部验证错误、payload、state_delta、候选工具调试文本不再显示给玩家。
- 角色台词只从结构化 `speech` / `dialogue_lines` 渲染，不再把旁白字段误当成角色气泡。
- 旁白与角色发言分离，公共叙事会移除嵌入式角色引号，避免旁白重复转述台词。
- AOHP 行动菜单支持紧凑目标选择，模型可以用动作编号加目标编号回答，减少“一人一个工具”导致的菜单膨胀。
- 工具失败反馈提供正确格式示例；重复的格式、目标、地点类失败会在当前回合后续菜单里隐藏。
- 动态工具路由改为全目录扫描和上下文评分，不再只从前几十个工具里抽候选。
- 目录工具目标推断不再用可见条件误判参数，只使用明确 target rule 或明确工具 ID 后缀判断。
- 动态菜单增加场景门控和保底：食堂、医务室、图书馆、金融、照护、救助、移动、生存等工具按场景和状态进入，不再被低价值社交工具挤掉。
- “私人房间/私人空间”相关验证失败改为 system-only 事件；普通移动菜单不再把他人私人房间当普通目标。
- 元工具、调试工具、旧抽象目录工具不再进入 agent 菜单，也不能被自由调用执行。
- 工具目录瘦身：纯情绪、喜好、态度、寒暄、气氛动作等表达型工具不再占用菜单；表达内容改由普通说话、附近发言、私人笔记或记忆记录承接。
- 保留会改变世界状态的工具路径，包括移动、生存、照护、医疗、工作、金融、法律、犯罪、关系确认、生育流程和狼人杀流程。
- 左侧栏新增 live snapshot 链路，世界、居民、地点在一个快照里读取；手动刷新按钮直接重拉该快照。
- 前端关键快照分批落地，世界、居民、地点、事件不再被叙事、指标、选中角色详情等慢接口阻塞。
- WebSocket 刷新改为节流加 pending 队列，避免连续事件导致世界时间、地点人数、居民位置长期不刷新。
- 影响世界面板展开/收起状态修复，展开后不再渲染右下角能力快捷按钮。
- LLM 配置支持全局和单 agent 的流式开关、temperature、top_p、max_tokens、presence_penalty、frequency_penalty。
- OpenAI-compatible 客户端支持 SSE 流式响应。
- OpenAI-compatible 请求超时改为可配置，不再硬编码 60 秒；世界创建、解说、身份生成、运行中单角色 LLM 更新和人员配置导出均接入该字段。
- 狼人杀角色分配、阶段同步、讨论、投票、夜间能力、狼人一致夜袭、预言家/验尸官/守卫能力、观察者身份可见和投票历史接入调度器。
- 狼人杀讨论、投票、夜间阶段菜单收窄到阶段必要工具，减少吃饭、工作、普通社交抢走回合。
- 狼人杀圆桌发言改为主持式流程，当前发言人至少说一次后才能结束，格式失败时有自然发言兜底。
- 狼人杀投票规则支持第 1 天不放逐，第 2 天起必须放逐一名幸存者。
- 狼人杀夜晚结束后给存活或临界玩家做隐式夜间恢复，避免第二天圆桌被饥饿、脱水、疲劳和昏迷吞掉。
- 圆桌和投票等主持阶段会稳定存活玩家状态，并同步到讨论地点，避免存活玩家因昏迷或地点状态跳过发言。
- 人员配置导入能识别 zip 内容和带额外后缀的 `.zip.` 文件，不再把 zip 文件误当 JSON 解析。
- 记忆提示词选择混合重要长期记忆与近期事实，并按时间顺序呈现；关系、孩子、死亡、危机、承诺、金融和狼人杀推理事实优先。
- 婴儿/幼儿照护、医疗场景、求食物/水帮助、金融资讯读取等关键工具可达性增强。
