# 二字情绪 → 数值增减 规范（EMOTION_EFFECTS_SPEC）

本规范用于填写 `backend/app/data/emotion_effects.json`。该文件把约 3433 个**两字情绪词**映射到一组**软情绪数值**的增减。运行时：agent 说话/吃东西可选地用「情绪：XX」表达两个字的情绪，系统据此微调它的数值；agent 本身**不知道**这会改数值，只是自然表达。

## 数据格式

```json
{
  "开心": {"mood": 8, "fun": 4},
  "难过": {"mood": -8},
  "某情绪": {}
}
```

- 每个 key 是一个**两字中文情绪词**（已固定，请勿增删 key）。
- value 是该情绪触发时的数值增减。**空对象 `{}` 表示暂无效果**（运行时直接跳过，不报错）。
- 允许的字段如下（其它字段会被忽略）：

| 字段 | 含义 | 取值方向 |
|---|---|---|
| `mood` | 整体心情（开心↔难过） | 正=更开心，负=更低落。范围会被夹在 -100..100 |
| `stress` | 紧张/压力（焦虑、愤怒、恐惧↑；放松、平静↓） | 正=更紧张，负=更放松。0..100 |
| `social` | 社交连接感（温暖、相爱、亲密↑；孤独、疏离↓） | 正=更有连接，负=更孤独。0..上限 |
| `fun` | 趣味/刺激（兴奋、好玩↑；无聊、乏味↓） | 正=更有趣，负=更无聊。0..上限 |
| `affection` | **对说话/行动所指向的那个人**的好感度 | 正=对 TA 更有好感（喜欢/感激/暖心），负=对 TA 反感（愤怒/厌恶）。仅当本次行动有明确目标(visible_ref)时才生效；没有目标就忽略 |

> `mood/stress/social/fun` 作用在**说话者自己**身上；`affection` 作用在**说话者→对象**的关系上。一个情绪可以同时含两类字段（例如「暖心」既让自己心情好，也提升对对方的好感）。
> 未来若要扩展更多可影响的维度（如 trust 信任、fear 恐惧、conflict 冲突等关系字段），在 `backend/app/content/emotion_effects.py` 的 `EMOTION_ALL_FIELDS` 与 effect_engine 的施加逻辑里登记后即可同样按本规范填写。

## 取值幅度指引（重要：每次表达都会施加一次，务必保守）

- 一般情绪：每个字段 **±3 ~ ±8**。
- 强烈情绪（绝望、暴怒、狂喜、心碎、恐惧）：可到 **±9 ~ ±12**。
- 多数情绪只需写 **1~2 个字段**；不要四个字段都填。
- 没有把握的、过于抽象或中性的词，留 `{}`。

## 映射原则（按情绪大类）

- **快乐/正面**（开心、愉快、欣喜、满足、幸福）：`mood+`，活跃类再给 `fun+`，归属类给 `social+`。
- **放松/平静**（安心、惬意、宁静、淡定）：`stress-`，可带小 `mood+`。
- **爱/连接**（爱意、心动、亲密、暖心）：`social+` 为主，`mood+` 为辅。
- **思念/牵挂**（想念、思念）：轻微 `mood-` + `social-`（想念意味着此刻不在身边）。
- **悲伤**（难过、伤心、悲痛、失落、绝望）：`mood-` 为主，重度可带少量 `stress+`。
- **孤独**（孤独、寂寞、空虚）：`social-` 为主 + `mood-`。
- **愤怒**（生气、愤怒、暴怒、烦躁）：`stress+` 为主 + `mood-`，烦躁带 `fun-`。
- **恐惧/焦虑**（害怕、恐惧、焦虑、紧张、不安）：`stress+` 为主，重度带 `mood-`。
- **无聊**（无聊、乏味、厌倦、枯燥）：`fun-` 为主，厌倦带 `mood-`。
- **厌恶**（恶心、厌恶、嫌弃）：`mood-` + 小 `stress+`。
- **食物口味**：好吃/美味/可口/鲜美/香甜 → `mood+`（可带小 `fun+`）；难吃/难喝/苦涩 → `mood-`。
- **疲惫**（疲惫、疲倦、困倦）：轻微 `mood-`（体力另有系统处理，不要动 energy）。

## 注意

- 不要引入 mood/stress/social/fun 以外的字段（如 satiety/energy）——那些由具体行动（吃饭/睡觉）处理，避免重复计数。
- 同义词应给**相近**的数值，便于一致体验。
